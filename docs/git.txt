//Tips

Agregar commit
git commit -m "Agregar formulario de login"
git commit -m "Validación de correo en backend"
git commit -m "Corrección de bug en autenticación"

Puedes usar algo como Conventional Commits:

feat: nueva funcionalidad
fix: corrección de errores
refactor: mejoras internas sin cambiar funcionalidad
docs: documentación
test: pruebas
chore: tareas generals

git commit -m "feat: agregar módulo de usuarios"
git commit -m "fix: corregir error en login"
git commit -m "refactor: simplificar lógica de autenticación"

//Usa ramas para cada funcionalidad
git checkout -b feature/login

//Terminando
git checkout main

//Realizar merge
git merge feature/login


//EStatus de commits
git status

//Diferencias
git diff

//Eliminar rama una vez terminado de usar
git branch -d feature/login
